# 快速開始指南與檔案清單
# Quick Start Guide & File Manifest

## 🎯 5 分鐘快速開始

### 步驟 1: 下載並進入專案目錄
```bash
# 進入包含所有檔案的目錄
cd reagent-system
```

### 步驟 2: 執行應用程式

**Windows 用戶:**
```bash
# 雙擊 run.bat 或
run.bat

# 或在命令提示符中執行
python main.py
```

**Linux/macOS 用戶:**
```bash
# 給腳本執行權限
chmod +x run.sh

# 執行
./run.sh

# 或直接執行
python main.py
```

### 步驟 3: 登入系統

```
使用者名稱: admin
密碼: admin123
```

✅ **完成！系統已啟動**

## 🔑 關鍵快捷操作

### 常見操作步驟

#### 1️⃣ 新增試劑
1. 點擊「試劑設定」
2. 點擊「新增試劑」
3. 填入試劑資訊
4. 點擊「保存」

#### 2️⃣ 入庫試劑
1. 點擊「入庫」
2. 選擇試劑名稱
3. 輸入 LOT 號和有效期限
4. 輸入數量
5. 點擊「入庫」
6. 列印自動生成的標籤

#### 3️⃣ 出庫試劑
1. 點擊「出庫」
2. 掃描或輸入院內編號
3. 系統自動帶出試劑信息
4. 輸入出庫數量
5. 點擊「出庫」

#### 4️⃣ 查看庫存
1. 點擊「查詢」
2. （可選）輸入查詢條件
3. 點擊「查詢」
4. 查看結果表格

#### 5️⃣ 盤點試劑
1. 點擊「盤點」
2. 在表格中輸入實際庫存
3. 系統自動計算差異
4. 點擊「確認」保存

#### 6️⃣ 報廢試劑
1. 點擊「報廢」
2. 選擇試劑和 LOT 號
3. 輸入報廢數量和原因
4. 點擊「執行報廢」

#### 7️⃣ 查詢批號追溯
1. 點擊「追溯」
2. 輸入院內編號或 LOT 號
3. 點擊「查詢」
4. 查看完整的批號歷史

#### 8️⃣ 匯出報表（管理員）
1. 進入「報表」標籤
2. 選擇報表類型
3. 點擊「匯出 Excel」或「匯出 PDF」
4. 選擇保存位置

## 💾 資料庫備份與還原

### 自動備份
系統每日凌晨 2:00 自動備份至 `backups/` 目錄

### 手動備份
```bash
# 簡單複製資料庫檔案
cp reagent_system.db reagent_system.db.backup

# 或使用 SQLite 工具
sqlite3 reagent_system.db ".backup reagent_system.db.backup"
```

### 還原資料庫
```bash
# 直接覆蓋現有檔案
cp reagent_system.db.backup reagent_system.db

# 或使用 SQLite 工具
sqlite3 reagent_system.db ".restore reagent_system.db.backup"
```

## 🔐 使用者管理

### 建立新使用者（管理員功能）
1. 進入「系統設定」標籤
2. 點擊「新增使用者」
3. 輸入：
   - 使用者名稱（唯一）
   - 密碼（至少6字符）
   - 真實姓名
   - 角色（admin/user）
   - 部門（選填）
4. 點擊「保存」

### 修改密碼
1. 進入「系統設定」標籤
2. 找到使用者
3. 點擊「修改密碼」
4. 輸入新密碼
5. 點擊「保存」

### 使用者權限

**管理員 (admin):**
- ✅ 所有試劑操作
- ✅ 入庫/出庫/盤點/報廢
- ✅ 使用者管理
- ✅ 報表生成
- ✅ 系統設定
- ✅ 審計日誌

**一般使用者 (user):**
- ✅ 入庫/出庫
- ✅ 庫存查詢
- ✅ 盤點操作
- ❌ 無法新增/修改試劑
- ❌ 無法刪除資料
- ❌ 無法進行系統設定

## 📊 首頁儀表板說明

首頁顯示即時統計資訊：

### 統計卡片
- **庫存總數**: 所有試劑的總庫存量
- **今日入庫**: 今日新入庫的試劑數量
- **今日出庫**: 今日出庫的試劑數量
- **快過期**: 90天內即將過期的試劑數
- **低庫存**: 庫存少於安全庫存的試劑數

### 快過期試劑表 (90天內)
顯示即將過期的試劑列表：
- 🟢 綠色：正常（>90天）
- 🟡 黃色：警示（30-90天）
- 🔴 紅色：警急（0-30天）
- 🔴 深紅：已過期

### 低庫存警示
顯示庫存不足的試劑：
- 紅色標示缺少的數量

## 📈 系統提示

### 常見提示訊息

| 訊息 | 原因 | 解決方案 |
|------|------|--------|
| "使用者名稱或密碼錯誤" | 認證失敗 | 確認使用者名稱和密碼 |
| "試劑代碼已存在" | 重複的試劑 | 使用不同的試劑代碼 |
| "庫存不足" | 出庫數量超過現有 | 檢查實際庫存 |
| "LOT號不能為空" | 輸入驗證失敗 | 輸入有效的 LOT 號 |
| "有效期限格式錯誤" | 日期格式不正確 | 使用 YYYY-MM-DD 格式 |
| "您沒有權限" | 權限不足 | 聯繫管理員升級權限 |

## 🔧 進階配置

編輯 `config.py` 自訂系統行為：

```python
# 警示閾值
EXPIRY_WARNING_DAYS = 90        # 90天內黃色警示
EXPIRY_CRITICAL_DAYS = 30       # 30天內紅色警急

# 備份設定
AUTO_BACKUP_TIME = '02:00'      # 每日凌晨2點自動備份
BACKUP_RETENTION_DAYS = 30      # 保留30天備份

# 標籤列印尺寸
BARCODE_WIDTH = 50              # 寬度（毫米）
BARCODE_HEIGHT = 30             # 高度（毫米）

# 每頁顯示項目數
ITEMS_PER_PAGE = 20

# 日誌級別
LOG_LEVEL = 'INFO'              # DEBUG, INFO, WARNING, ERROR, CRITICAL
```
